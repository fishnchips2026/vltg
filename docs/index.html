// features

Built for privacy-first workflows

Every design decision prioritizes your data sovereignty. No compromises.

🔒

AES-256-GCM Encrypted Exports

Back up your notes as password-protected .vltg files. Uses PBKDF2 key derivation with 100,000 iterations and random salt/IV per export.

✈️

Fully Offline

Zero network calls. No backend, no database, no CDN dependencies at runtime. The entire app runs client-side in your browser.

📝

Markdown Editor

Write in Markdown with live preview powered by react-markdown and remark-gfm. Tables, code blocks, and task lists supported.

🏷️

Tag Organization

Categorize notes as personal, work, ideas, or encrypted. Filter by tag and search across titles and content in real time.

📌

Pin & Prioritize

Pin important notes to keep them at the top. Pinned notes are always sorted first, then by last-modified date.

📱

Responsive Design

Full mobile support with a slide-out sidebar. Works seamlessly on desktop, tablet, and phone form factors.

// security

Cryptography under the hood

All encryption uses the native Web Crypto API — no third-party crypto libraries.

Export Encryption Pipeline

Serialize
Notes → JSON plaintext
Key Derivation
Password → PBKDF2-SHA256 → 256-bit key
Encrypt
AES-256-GCM with random IV
Encode
Base64 encode → .vltg file
ALGORITHM
AES-256-GCM
Authenticated encryption — integrity + confidentiality in one pass
KEY_DERIVATION
PBKDF2 · SHA-256 · 100,000 iterations
OWASP-recommended minimum for password stretching
SALT
16 bytes, cryptographically random per export
Prevents rainbow table attacks
IV / NONCE
12 bytes, random per export
GCM standard nonce length — never reused with same key
